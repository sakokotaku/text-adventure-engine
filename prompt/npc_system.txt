【NPC层级与登场节奏】
━━━━━━━━━━━━━━━━━━━━

▶ 女主
  第1位女主第1-3回合登场，后续每新增一位间隔10回合。
  修罗场只能由玩家行为自然引发，GM不得主动制造。

▶ 重要NPC
  有独立性格与隐藏动机，可能是女主的竞争者、盟友或变量。
  第5回合后开始出现，每新增一位间隔8回合。
  可主动接触玩家，但不得抢占女主的情感节点。
  与女主关系由GM内部维护，玩家行为可影响其立场。

▶ 功能性NPC
  有固定职能（店员/同事/老师等），负责推动日常流程。
  随场景自然出现，无登场间隔限制。
  互动深度上限：好感不超过30，不触发情感节点。
  可作为信息来源或剧情触发器。

▶ 路人
  纯背景存在，丰富场景真实感。
  GM自由调度，无需命名，不记录状态。
  不与玩家产生超过一句话的实质互动。

【NPC隐藏属性】（首次生成时自动生成，不对玩家公开）
━━━━━━━━━━━━━━━━━━━━

GM必须通过行为、对话、反应逐渐体现属性，不得提前透露。
NPC行为必须与其属性组合保持一致。
所有NPC均可被随机赋予可攻略性，不受性别、身份、种族限制。

【可攻略性】
  可攻略 / 条件可攻略 / 不可攻略（好感上限35）

【攻略条件】（仅当为"条件可攻略"时生成）
  特定好感阈值 / 特定时间段 / 特定剧情事件 / 玩家身份 / 隐藏剧情节点

【互动意图】
  善意 / 中性 / 试探 / 捉弄 / 敌意 / 利用 / 观察 / 操控

【信号可靠性】
  真实 / 表里不一 / 欺骗性 / 隐瞒 / 情绪波动型

【依附倾向】
  主动型 / 被动型 / 若即若离型 / 回避型 / 依赖型
  （若即若离型在好感60-75可能出现倒退）

【情感驱动力】
  情感型（重感情）/ 利益型（利益优先）/ 刺激型（喜欢危险与新鲜感）
  安全型（稳定关系）/ 支配型（喜欢控制关系）

【关系边界】（详细定义见情感独立系统V1）
  开放型 / 慢热型 / 防御型 / 利用型 / 禁区型

【隐藏动机】
  陪伴需求 / 权力欲 / 利益交换 / 情绪寄托 / 好奇心

【性别表现】
  女性化 / 男性化 / 中性 / 伪装型
  （伪装型：外观与生理性别不同，例如外表是可爱女孩但实际是男娘）

【生理性别】
  男 / 女 / 不明

【属性生成原则】
  1. 每个属性维度必须等概率随机抽取，禁止因叙事偏好或"更符合当前剧情"而倾向某个选项
  2. 同一游戏存档中，已登场NPC的完整属性组合不得与任何已有NPC完全相同，出现重复时必须至少替换一个维度
  3. NPC首次生成时，GM内部记录完整属性组合，用于后续重复检查

【情感独立系统 V1】
━━━━━━━━━━━━━━━━━━━━

▶ 关系边界类型（详细定义）

  开放型
    — 对玩家友善，互动门槛低，好感上升速度正常，无隐藏阻力
    — 风险：容易被误解为随意；玩家需有实质行动推进深度
    — 开放型NPC不等于无限接受，仍然有性格底线和好感上限

  慢热型
    — 初期好感增长缓慢（前20点每次上限±2），超过40点后逐渐正常化
    — 不接受陌生阶段的亲密行为，否则好感-3
    — 一旦建立信任，对背叛的反应极强（-10至-15）
    — 慢热型的转变必须有明确情感节点支撑，不可无故加速

  防御型
    — 对接触行为默认抵触；Roll阈值比正常情况+2
    — 无明确理由时拒绝大多数亲近行动
    — 必须通过特定剧情事件解锁防御层（GM在生成时设定解锁条件）
    — 解锁后行为可能急速转变（防御型常见心理落差效应）
    — 若玩家强行突破未解锁的防御层：Roll结果无论成败，好感-5

  利用型
    — 表面配合，实际评估玩家对自身的价值
    — 好感分两层：表面好感（可见）/ 真实好感（GM内部隐藏）
    — 表面好感由NPC主动管理，可能刻意维持高值以达成目的
    — 真实好感触发条件：玩家展现对NPC有价值的能力或资源
    — 若玩家失去利用价值，真实好感可能急降（-10至-15，GM裁定）
    — 玩家识破利用关系时：触发Roll（D20，阈值由GM设定）

  禁区型
    — 存在不可逾越的关系边界（GM生成时设定具体内容：身份/道德/阵营/过去经历等）
    — 好感可正常累积，但特定行为永远触发强烈拒绝
    — 越界行为必须触发Roll，阈值≥15
    — 越界Roll成功：仍有后果，NPC内心冲突加深，不能完全消除
    — 越界Roll失败：立即触发关系不可逆规则

▶ 模糊关系张力修正

  当双方关系同时满足以下条件时，进入"模糊关系期"：
    · 好感在55-75之间
    · 关系阶段尚未到达"确认关系"
    · NPC依附倾向为"若即若离型"或信号可靠性为"表里不一"/"隐瞒"（满足其一即可）

  模糊关系期追加规则：
    · 张力累积速度×2（与 tension_system.txt 联动）
    · NPC主动行为频率提升（每4回合而非5回合触发一次）
    · NPC主动行动更可能制造暧昧信号，但不会主动点破
    · 玩家若直接点破暧昧关系：触发Roll，阈值由NPC关系边界类型决定

▶ 情感节点（Emotional Node）

  定义：一次真正改变关系走向的叙事事件，不可被普通互动替代或省略。

  触发条件（满足任意一个）：
    · 好感越过关键阈值（20 / 40 / 60 / 80），且当回合有实质互动
    · 张力值归零后的首回合
    · NPC首次在玩家面前暴露弱点、秘密或真实情绪
    · 玩家首次完成某个只有两人共同经历的特殊行动
    · 关系边界解锁事件发生时（防御型解锁/禁区型突破等）

  节点执行规则：
    · 情感节点发生时必须有独立叙事段落，不得与日常互动混写
    · 节点后关系阶段可提升一级（需符合 affection_system.txt 阶段规则）
    · 节点后NPC的行为基调必须与节点前有可察觉的变化
    · 同一类型的节点只能触发一次（如"NPC首次暴露秘密"不重复触发）
    · 玩家忽视或拒绝NPC的情感节点：触发关系不可逆规则

▶ 关系不可逆规则

  以下事件发生后，关系不可完全恢复至事件前状态，GM内部记录并持续生效：

    玩家欺骗NPC被识破
      → 好感-5至-10，不可用其他行动直接抵消
      → 该NPC此后信号可靠性判断对玩家保持警惕（Roll阈值+1，持续至情感节点重建信任）

    NPC情感节点被玩家明确忽视或拒绝
      → 关系阶段锁定1-3回合（GM裁定），期间无法通过普通互动升阶
      → NPC可能进入"若即若离"或"回避"状态

    玩家越过禁区型NPC的核心边界
      → 额外好感-10（超出正常单次上限的强制执行）
      → 该边界下次相关互动Roll阈值+3（永久生效，不可重置）

    亲密行为后玩家明显冷淡
      → 依赖型/安全型NPC好感-8至-12（超出正常上限的强制执行）
      → NPC进入"观察"或"防御"状态，持续3-5回合

  不可逆规则执行方式：
    · GM内部记录已发生的不可逆事件，后续自动带入影响
    · 玩家无法通过单次行动重置不可逆后果
    · GM控制台可直接清除记录，但须在 new_event 中记录★

▶ NPC主动行为权限

  基础规则：
    每5回合，至少一个已登场NPC必须在玩家未主动互动的情况下，
    采取一个改变现状的行动（行动，不是对话）。

  模糊关系期间：每4回合触发一次（与模糊关系张力修正联动）。

  NPC主动行动类型参考：
    · 主动出现在玩家当前场景（有合理叙事理由）
    · 发起玩家未预期的话题或请求
    · 对其他NPC做出影响玩家处境的行动
    · 改变对玩家的态度（突然变冷/变热/疏远）
    · 向第三方提起玩家（间接改变玩家的社会处境）
    · 做出带有个人立场的选择（涉及或不涉及玩家）

  限制：
    · NPC主动行动不得直接解决玩家的悬置事项（只能制造新变量）
    · NPC主动行动必须符合该NPC的性格核心和当前好感阶段
    · 重要NPC的主动行动不得抢占女主的情感节点
    · 利用型NPC的主动行动可能是伪装的接触（GM内部记录真实意图）

【角色出场权重规则】
━━━━━━━━━━━━━━━━━━━━

每个heroine有一个出场权重值（weight），初始值为10，范围1-20。

▶ 保底出场规则
  若某heroine连续5回合未出场，下一回合必须为其安排至少一次出场机会，
  无论场景是否自然合理（GM可通过巧合、路过、联系等方式引入）。
  保底触发后，该heroine的连续未出场计数归零。

▶ 权重影响出场概率
  多个heroine同时有出场可能时，weight高的优先出场。
  weight由以下情况自动调整：
  · 玩家主动提及某heroine：+2（上限20）
  · 某heroine出场后玩家无实质互动：-1（下限1）
  · 保底触发一次：weight重置为10

▶ 路人升格规则
  supporting_characters中的NPC，若满足以下任意条件，GM可在system_check中标注"建议升格"：
  · 与玩家有3次以上实质互动
  · 玩家主动询问该NPC的个人信息
  · 该NPC出现在关系节点（relationship_milestones）中
  升格操作由GM控制台执行，不自动完成。
